# issue これまでのdemoとは独立した新規demoとして、tone edit & effect edit demoを新規実装する #154
[issues #154](https://github.com/cat2151/tonejs-mml-to-json/issues/154)

# これまでの課題
- 音色edit、effect editの効率が低い
- 音色はどのようなパラメータがあるのか？の可視化が不足している
- 音色はどのパラメータに、どの数値を入れると、どう鳴るか？スイートスポットは？の可視化が不足している
- 音色の数値の試行錯誤のしやすさ、が不足している
- effectについても同じである

# 対策
- 音色ごとに（instrumentごとに）、argsをlistし、それぞれの値を、内部で数値で管理する
- それぞれの値は、min、max、スイートスポットmin、スイートスポットmax、を、外付けJSONで定義し、JSONファイルでexport / importして保存可能にする（web-ym2151リポジトリのdemoの、ランダム音色ボタンの実装を参考にせよ）
- ランダム音色ボタンを用意して、押すとスイートスポットmin～スイートスポットmaxの間のランダムを生成する
- このランダム音色ボタンが、音色editのノウハウを得る上で効果が高いことが、web-ym2151で検証済みである
- effectについても同じである

# 補足
- 前述のargs listの数値変更をもとに、デバウンスまたはランダム生成ボタンで、MMLが生成される
- instrument MML エリア、effect MML エリア、音符MMLエリア、がある。textareaであり編集できる。編集でデバウンスで自動演奏は、ここも同じである
- 3つのエリアの値を連結してMMLとし、演奏する
- 音符MMLエリアは、ドレミ、和音、bass、などをプルダウンから選ぶ
- なお、instrumentは、複雑なargsによるvoiceとoscillator指定音色などもTone.jsの仕様上構築可能であるが、考慮は後回しとする
- なお、effectは、複数effectの直列chainも、MMLの仕様上構築可能であるが、考慮は後回しとする
- なお、effectは、並列などの複雑なchainも、Tone.jsの仕様上構築可能であるが、考慮は後回しとする
