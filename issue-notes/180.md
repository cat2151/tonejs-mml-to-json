# issue tone edit & effect edit demo の画面レイアウトを改善する #180
[issues #180](https://github.com/cat2151/tonejs-mml-to-json/issues/180)

- ターゲットは1920x1080のブラウザ
- 上下左右のスキマが多すぎてUXが悪い。例えばEffectという見出しの上下がスキマがムダすぎる。狭くする
- それぞれの横幅が広すぎてUXが悪い。今の半分にする
- デバウンスが長すぎてUXが悪い。まずデバウンスなしで検証とする
- JSONエラーがあると不正な音で鳴ってしまいUXが悪い。すべてのJSONエラーはstatus表示エリアに表示し、演奏ナシとする
- ボタン、ランダムボタンとランダムsweet spotボタンがあり煩雑でUXが悪い。ランダム音色ボタンに絞り、その内容はsweet spot版とする
- プルダウンの選択肢に欠落があり効果が薄いし、ランダムで選択できないのも意味がない。
  - プルダウンの内容を修正する。具体的にはInstrumentはMMLがサポートする全てをプルダウンに入れる。Effectも同様。
  - ランダム関数を新規追加し、プルダウンで選ぶのと同様のことを実現する。InstrumentとEffectそれぞれ対象。
- 項目名称を表示してボタンで上下させるのは効果が薄いし、ムダに複雑でUXが悪い。廃止する。
- Notesのドレミのプリセットが誤っている上にムダに複雑。「cdefgab<c」 に修正する
- Notesの和音のプリセットがダサい上にムダに複雑。「'ceg'」 にする
- NotesのMMLにoを書くのをやめて、octave up 「<」か、down「>」で表現する
- JSON (tonejs-json-sequencer) 欄が編集できないと検証しづらい。textareaにし、デバウンス0とする
- すべてのtextareaは、デバウンス0で、エラーはstatus表示エリアに表示し、演奏ナシとする
