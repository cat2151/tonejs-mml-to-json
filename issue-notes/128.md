# issue 例えばFMSynthでシングルクォートの和音コマンドを使うと、PolySynthになったあとFMSynth部分が忘れられてしまう #128
[issues #128](https://github.com/cat2151/tonejs-mml-to-json/issues/128)

# 考察
- 対策として、
    - 和音コマンドによるPolySynth化の際に、
    - PolySynthにargsを指定し、
    - そのargsは、
        - voice FMSynthと、
        - options （中身はFMSynthのargs）にすべきである
    - 詳しくは、tonejs-json-sequencer リポジトリの demo の PolySynth（FM Synth） を参照のこと
        - ただし現在、同じJSON（前述の `tonejs-json-sequencer リポジトリの demo の PolySynth（FM Synth）`）を、それぞれで演奏したときに、tonejs-json-sequencer なら演奏できるのに、当リポジトリだと演奏できない、という問題が発生している。
            - エラーログ : `tonejs-json-sequencer.mjs:212 Error creating node or connection: TypeError: t.voice.getDefaults is not a function    at new Xr (PolySynth.ts:153:34)    at createNode (tonejs-json-sequencer.mjs:98:39)    at scheduleOrExecuteEvent (tonejs-json-sequencer.mjs:40:13)    at tonejs-json-sequencer.mjs:208:17    at Array.forEach (<anonymous>)    at playSequence (tonejs-json-sequencer.mjs:205:14)    at play (play.ts:60:11)    at HTMLTextAreaElement.handleManualInput (main.ts:93:5)`
            - 手短に調査すること。
                - 無理なら別issueに切り出すので、無理かどうかだけ報告すること。
                - おそらくだが、別issueに起票済みの、「古いバージョンのライブラリを使い続けていること」が原因の可能性がある
